# ADR 001: Wybór Next.js App Router i lokalnej bazy faktów

## Status
Zatwierdzony

## Kontekst
Projekt ma w założeniu być prostą stroną internetową do prezentacji losowych ciekawostek kognitywistycznych. Wymagane jest szybkie wdrożenie, łatwy rozwój i ograniczona złożoność infrastruktury.

## Decyzja
Wybieramy Next.js z App Router jako podstawę aplikacji oraz lokalny plik danych `data/facts.ts` jako główne źródło treści. Dodatkowo zachowujemy endpoint API `/api/fact` do opcjonalnej integracji z AI.

## Uzasadnienie
- Next.js App Router oferuje dobry balans między generowaniem statycznym a serwerowym renderowaniem.
- Lokalna baza faktów zapewnia niezawodność, szybkość i prostą walidację danych.
- Integracja AI jest opcjonalna i może być użyta jako warstwa dodatkowa, nie jako jedyne źródło treści.

## Konsekwencje
- Aplikacja może działać offline z lokalną bazą faktów.
- API AI wymaga konfiguracji `GOOGLE_AI_KEY`, ale awaria tego komponentu nie blokuje podstawowej funkcji.
- Rozszerzenia w przyszłości mogą dodać bazę danych, ale na etapie MVP nie są potrzebne.
