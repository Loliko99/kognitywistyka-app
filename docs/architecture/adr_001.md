# ADR 001: Next.js App Router i lokalna baza faktów

## Status
Zaakceptowany

## Kontekst
Projekt ma być prostą, stabilną aplikacją edukacyjną. Wymagane są szybkie wdrożenie, czytelny routing, możliwość generowania stron szczegółów oraz zgodność z podejściem Spec Driven Development.

## Decyzja
Używamy Next.js z App Routerem jako podstawy aplikacji. Statyczne ciekawostki przechowujemy w `data/facts.ts`, a kontrakt danych definiujemy w `domain/fact.ts`.

## Uzasadnienie
- Next.js daje routing, API routes i wdrożenie na Vercel bez dodatkowego backendu.
- Lokalna baza faktów jest prosta, audytowalna i wersjonowana w repozytorium.
- TypeScript pozwala utrzymać spójność danych między komponentami, API i walidacją.

## Konsekwencje
- Projekt jest łatwy do uruchomienia i sprawdzenia lokalnie.
- Repozytorium pozostaje źródłem prawdy dla statycznych treści.
- Brak bazy danych oznacza, że treści AI nie są trwale zapisywane.
