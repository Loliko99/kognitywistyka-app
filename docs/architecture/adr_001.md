# ADR 001: Next.js App Router i lokalna baza faktów

## Status
Zaakceptowany

## Kontekst
Projekt wymaga prostego wdrożenia, dynamicznych podstron faktów, endpointu API oraz możliwości rozwoju zgodnie z SDD.

## Decyzja
Używamy Next.js App Router. Statyczne fakty przechowujemy w `data/facts.ts`, a kontrakt domenowy w `domain/fact.ts`.

## Uzasadnienie
- App Router obsługuje zarówno landing page, jak i dynamiczne `/fact/[id]`.
- Lokalna baza faktów jest wersjonowana w repozytorium i zgodna z zasadą single source of truth.
- TypeScript pozwala utrzymać spójność danych.

## Konsekwencje
- Fakty statyczne są łatwe do audytu.
- Zmiana statycznych treści wymaga commita.
- Dynamiczne fakty AI wymagają osobnej warstwy zapisu, jeśli mają mieć trwałe linki.
