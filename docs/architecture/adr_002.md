# ADR 002: Generowanie ciekawostek AI jako funkcja opcjonalna

## Status
Zaakceptowany

## Kontekst
Aplikacja ma generować nowe ciekawostki, ale nie może przestać działać przy problemach z Gemini, kluczem API, limitem usługi albo niepoprawnym JSON-em.

## Decyzja
Generowanie AI realizujemy w `app/api/fact/route.ts`. Endpoint działa po stronie serwera, używa `GOOGLE_AI_KEY`, wymusza JSON, waliduje dane i ma fallback do lokalnej bazy.

## Uzasadnienie
- Klucz API nie trafia do frontendu.
- Błąd zewnętrznego dostawcy nie blokuje strony.
- Walidacja chroni UI przed niepełną odpowiedzią modelu.

## Konsekwencje
- Endpoint jest miejscem kontroli jakości odpowiedzi AI.
- Bez poprawnego Gemini użytkownik nadal otrzymuje fakt z lokalnej bazy.
- Funkcja wymaga testów parsera i fallbacku.
